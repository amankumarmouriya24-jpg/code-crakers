# Security Policy

## Supported Versions

The latest `main` branch and the most recent tagged release receive security updates.

## Reporting a Vulnerability

Please report security issues privately to the repository maintainer instead of opening a public issue.

Include:

- A clear description of the vulnerability
- Steps to reproduce it
- Affected files or routes
- Any suggested fix, if known

The maintainer will review the report, confirm impact, and publish a fix as soon as practical.

## Security Practices

- Do not commit secrets, tokens, credentials, or private keys.
- Use `.env.example` to document required environment variables.
- Run dependency and static-analysis checks before release.
